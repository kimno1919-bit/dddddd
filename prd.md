> **문서 버전**: v1.0
> **작성일**: 2026-06-27
> **상태**: Draft
> **타입**: Product Requirements Document

---

## 1. 개요 (Overview)

### 1.1 제품명
**LearnHub** (가칭) — 개인이 제작한 학습용 웹앱을 한곳에 모아 빠르게 실행할 수 있는 앱 런처형 포털.

### 1.2 배경
- 여러 학습용 웹앱을 개별 URL/북마크로 관리하면 접근성이 떨어지고, 어떤 앱이 있는지 한눈에 파악하기 어렵다.
- macOS Launchpad, ChromeOS 앱 런처, Notion 대시보드처럼 **한 화면에서 모든 앱을 탐색·실행**할 수 있는 허브가 필요하다.

### 1.3 목표
1. 보유한 모든 학습용 웹앱을 **3초 이내**에 찾아 실행할 수 있다.
2. 카테고리/검색으로 앱 탐색이 직관적이다.
3. 새 앱을 추가하는 비용이 **JSON 한 줄 수준**으로 낮다.

### 1.4 비목표 (Non-goals)
- 사용자 인증/회원가입 (1차 범위 제외)
- 앱 내부 학습 진도 추적 (각 앱이 자체 관리)
- 다중 사용자 권한 관리

## 3. 핵심 기능 (Features)

### 3.1 화면 구성

세 가지 핵심 뷰를 **상단 탭**으로 전환한다.

| 뷰 | 목적 | 기본값 |
|---|---|---|
| **전체 앱** | 모든 앱을 한 그리드에 표시 | 기본 진입 화면 |
| **카테고리** | 카테고리별로 그룹화하여 표시 | — |
| **앱 런처** | macOS Launchpad 스타일 풀스크린 격자 | 단축키 진입 |

---

### 3.2 기능 명세

#### F-01. 전체 앱 뷰 (All Apps)

**설명**: 등록된 모든 웹앱을 카드 그리드로 표시.

**요구사항**:
- 반응형 그리드: 데스크톱 4~6열 / 태블릿 3열 / 모바일 2열
- 카드 구성: 아이콘, 앱 이름, 1줄 설명, 카테고리 뱃지
- 정렬 옵션: 최근 사용순(기본) / 이름순 / 추가일순
- 카드 클릭 → 새 탭으로 앱 실행 (`target="_blank"`)
- 상단 검색바: 앱 이름·설명·태그 실시간 필터링

**완료 기준 (Acceptance Criteria)**:
- [ ] 50개 앱 등록 시 스크롤 없이 한 화면에서 그리드 렌더링 가능
- [ ] 검색어 입력 시 200ms 이내 결과 갱신
- [ ] 카드 호버 시 마지막 사용일 툴팁 표시

---

#### F-02. 카테고리 뷰 (Categories)

**설명**: 카테고리별 섹션으로 앱을 그룹화하여 표시.

**요구사항**:
- 카테고리 예시: `영어`, `수학`, `코딩`, `과학`, `역사`, `기타`
- 각 섹션 헤더에 카테고리명 + 앱 개수 뱃지
- 좌측 사이드바: 카테고리 목록 + 클릭 시 해당 섹션으로 스크롤
- 카테고리는 JSON 메타데이터에서 자동 추출 (하드코딩 금지)
- 빈 카테고리는 표시하지 않음

**완료 기준**:
- [ ] 카테고리 추가 시 코드 수정 없이 사이드바·섹션 자동 생성
- [ ] 사이드바 클릭 시 부드러운 스크롤 이동
- [ ] 모바일에서는 사이드바가 상단 가로 칩으로 변환

---

#### F-03. 앱 런처 뷰 (App Launcher)

**설명**: macOS Launchpad / iOS 홈 화면 스타일의 풀스크린 격자형 런처.

**요구사항**:
- 화면을 덮는 풀스크린 모달 형태
- 큰 아이콘 + 짧은 이름만 표시 (밀도 ↑)
- 격자: 데스크톱 8열 / 태블릿 6열 / 모바일 4열
- 페이지네이션 또는 무한 스크롤 (앱이 많을 경우)
- 키보드 단축키:
  - `Cmd/Ctrl + K` → 런처 열기
  - `Esc` → 닫기
  - 방향키로 포커스 이동, `Enter`로 실행
- 상단에 검색 인풋 자동 포커스

**완료 기준**:
- [ ] 단축키로 어디서든 0.3초 이내 런처 호출
- [ ] 키보드만으로 앱 선택·실행 가능
- [ ] 배경은 블러 없이 반투명 오버레이 (성능 우선)

---

#### F-04. 검색

**설명**: 전역 검색으로 모든 뷰에서 앱을 빠르게 찾는다.

**요구사항**:
- 검색 대상: 앱 이름, 설명, 카테고리, 태그
- 부분 일치(substring) + 대소문자 무시
- 한글 초성 검색은 v1.1로 연기
- 검색 결과 0건 시 빈 상태(Empty State) 메시지 표시

---

#### F-05. 앱 메타데이터 관리

**설명**: 앱 등록·수정은 단일 JSON 파일(`apps.json`)로 관리.

**스키마**:
```json
{
  "apps": [
    {
      "id": "fraction-calc",
      "name": "분수 계산기",
      "description": "분수 연산을 시각화로 학습",
      "url": "https://example.com/fraction",
      "icon": "fraction.svg",
      "category": "수학",
      "tags": ["계산", "초등", "시각화"],
      "color": "#2563eb",
      "createdAt": "2026-05-01"
    }
  ]
}
```

**필수 필드**: `id`, `name`, `url`, `category`
**선택 필드**: `description`, `icon`, `tags`, `color`, `createdAt`

**완료 기준**:
- [ ] JSON에 항목 1개 추가 → 새로고침만으로 포털에 반영
- [ ] 필수 필드 누락 시 콘솔 경고 + 해당 앱 스킵

---

#### F-06. 개인화 (LocalStorage)

**설명**: 서버 없이 브라우저에 사용 이력을 저장.

**저장 항목**:
- `lastUsedAt[appId]`: 마지막 실행 시각 → 정렬에 활용
- `favorites[]`: 즐겨찾기 앱 ID 목록
- `viewPreference`: 마지막 선택한 뷰 (전체/카테고리/런처)

**완료 기준**:
- [ ] 앱 클릭 시 `lastUsedAt` 자동 갱신
- [ ] 즐겨찾기 토글 → 카드 우상단 별 아이콘 상태 변화
- [ ] LocalStorage 초기화 버튼 제공 (설정 영역)

## 4. 비기능 요구사항 (Non-functional Requirements)

| 항목 | 요구사항 |
|---|---|
| 성능 | 초기 로딩 1초 이내 (앱 100개 기준) |
| 반응형 | 320px ~ 2560px 폭에서 정상 렌더링 |
| 접근성 | WCAG 2.1 AA 준수, 키보드만으로 모든 기능 사용 가능 |
| 브라우저 | Chrome/Safari/Firefox/Edge 최신 2개 버전 |
| 오프라인 | PWA 지원 (v1.1) |
| 다크모드 | OS 설정 자동 감지 + 수동 토글 |

## 8. 기술 스택 제안

| 영역 | 권장 | 대안 |
|---|---|---|
| 프레임워크 | 순수 HTML/CSS/JS (단일 파일) | Vite + Vanilla JS |
| 데이터 | `apps.json` 정적 파일 | — |
| 호스팅 | GitHub Pages / Vercel | Netlify |
| 아이콘 | SVG 인라인 또는 이모지 | Lucide Icons |
| 빌드 | 불필요 (정적) | — |

**선택 근거**: 학습용 웹앱 포털은 동적 백엔드가 불필요. 정적 호스팅으로 무료·고속·유지보수 최소화.
